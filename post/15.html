ReactJS + Github-Pages com deploy automático
Destaque

Posted on April, 2020 bycfchenr

Introdução

Neste artigo vou explicar como podem correr uma aplicação ReactJS nas páginas do Github (<github user>.github.io) e automatizar o deploy sempre que houverem alterações no código fonte da aplicação.

Criar a aplicação ReactJS

O primeiro passo é criarmos a nossa aplicação ReactJS. Para tal executamos os seguintes comandos:

npx create-react-app my-app
cd my-app
npm start

Para verificar que está tudo a correr como esperado, acedemos ao url http://localhost:3000 e verificamos se a resposta da aplicação é idêntica à representada na seguinte figura:

De forma a conseguirmos ligar a aplicação ReactJS ao sistema de páginas do Github instalamos a dependência gh-pages:

npm install gh-pages --save-dev

Agora que temos a nossa aplicação a correr sem problemas vamos alterar algumas configurações essênciais no ficheiro package.json:

  • Adicionar a chave homepage:
//...
"homepage": "http://<github user>.github.io/my-app"
  • Aos scripts existentes adicionamos 2 novos, predeploy e deploy:
"scripts": {
  //...
  "predeploy": "npm run build",
  "deploy": "gh-pages -d build"
}

Criar o repositório no Github

Uma vez criada e configurada a aplicação ReactJS chega a altura de criarmos o nosso repositório onde vai estar o código fonte da aplicação.

git init
git add .
git commit -m "Initialize project using Create React App"
git remote add origin https://github.com/<github user>/my-app.git
git push -u origin master

Agora que temos todo o nosso código no repositório git é altura de fazer o deploy para as páginas do Github. Para isso vamos correr o seguinte comando:

npm run deploy

Quando esta etapa terminar podemos verificar no nosso repositório que foi criada uma nova branch chamada gh-pages. Isso é bom sinal! Agora podemos aceder ao url da página do Github (<github user>.github.io) e vemos a nossa aplicação a funcionar!

Automatizar a tarefa de deploy

Com a página do Github a funcionar passamos ao passo seguinte: realizar esta tarefa de deploy da nossa aplicação sempre que existir novo código. Para isso vamos recorrer às actions do Github.

Criar um Secret

Para que a tarefa seja realizada é necessário dar permissões de leitura no nosso repositório através de um Secret. Para tal basta:

  • Ir à página Developer settings no Github > Gerar um novo token > Ir até à página de Secrets do nosso repositório > Criar um novo Secret com o nome PERSONAL_TOKEN e colocar o valor obtido quando gerámos o token.

Criar um ficheiro na pasta .github/workflows/masterDeploy.yml (pode ter outro nome) com o seguinte conteúdo:

name: masterDeploy

on:
  push:
    branches:
      - master

jobs:
  deploy:
    runs-on: ubuntu-18.04
    steps:
      - uses: actions/checkout@v2

      - name: Setup Node
        uses: actions/setup-node@v1
        with:
          node-version: '10.x'

      - name: Cache dependencies
        uses: actions/cache@v1
        with:
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-node-

      - run: npm ci
      - run: npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          user_name: '<github user>'
          user_email: '<github email>'
          personal_token: ${{ secrets.PERSONAL_TOKEN }}
          publish_dir: ./build

Com esta configuração, sempre que houverem commits na branch master do repositório, vai ser realizada um novo deploy da aplicação.

Fazer o deploy para um repositório diferente

No caso de querermos fazer o deploy da aplicação para um repositório externo, de forma a separar o código fonte do código da aplicação compilada, podemos alterar o ficheiro do workflow da seguinte forma:

//...
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          user_name: '<github user>'
          user_email: '<github email>'
          personal_token: ${{ secrets.PERSONAL_TOKEN }}
          external_repository: <github user>/<github external repository>
          publish_branch: <branch>
          publish_dir: ./build

O website da minecode é um exemplo claro onde fazemos o deploy da aplicação para o repositório minecode.github.io a partir do código fonte no repositório website.
